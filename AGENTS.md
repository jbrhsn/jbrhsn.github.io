## Project Shape

- Bun project (`bun.lock`, `bunfig.toml`); prefer `bun run ...` / `bunx ...` over npm/pnpm/yarn.
- Main app is TanStack Start for preview/dev: `src/routes/index.tsx` renders `src/portfolio/Portfolio.tsx` using data from `src/portfolio/data.ts`.
- GitHub Pages uses a separate static SPA entry: `static/index.html` -> `src/entry-static.tsx` -> `Portfolio`; this bypasses TanStack Start and outputs to `dist-static/`.
- Most portfolio copy is centralized in `src/portfolio/data.ts`; edit there before hard-coding text in components.

## Commands

- Dev server: `bun run dev`.
- TanStack Start production build: `bun run build`.
- GitHub Pages/static build: `bunx vite build --config vite.static.config.ts`.
- Project-page base path: `VITE_BASE=/<repo>/ bunx vite build --config vite.static.config.ts`; default base is `./` for portable/static preview output.
- Lint: `bun run lint`.
- Format whole repo: `bun run format`.
- There is no test or typecheck script in `package.json`; use build/lint as the available verification unless you add scripts.

## TanStack / Vite Gotchas

- `src/routes/` is TanStack file-based routing. Do not introduce Next/Remix structure such as `src/pages/`, `src/routes/_app/index.tsx`, or `app/layout.tsx`.
- `src/routes/__root.tsx` is the only root layout; preserve `<Outlet />` or child routes stop rendering.
- `src/routeTree.gen.ts` is generated by TanStack Router; do not edit it by hand.
- Server entry is intentionally redirected in `vite.config.ts` to `src/server.ts`, which wraps TanStack Start SSR errors.

## Styling / UI

- Tailwind v4 is configured through `src/styles.css` with `@import "tailwindcss" source(none);` and `@source "../src"`; there is no separate Tailwind config file.
- The visual language is cyberpunk/HUD themed with custom utilities in `src/styles.css` (`hud-panel`, `neon-cyan`, `scanlines`, etc.); reuse those before adding generic card/layout styles.
- The whole site is hand-written custom markup in `src/portfolio/Portfolio.tsx` driven by `src/portfolio/data.ts`; there is no component library. The legacy shadcn `:root`/`@theme inline` tokens in `src/styles.css` are only used by the 404/error UI in `src/routes/__root.tsx`.

## Install / Dependency Safety

- `bunfig.toml` enforces `minimumReleaseAge = 86400`.
